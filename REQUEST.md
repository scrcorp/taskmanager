# 프로젝트: TaskManager 앱 구현 요청서

## 1. 개요
제공된 프로토타입과 API 명세서(V2)를 바탕으로 **Flutter**를 사용하여 실제 작동하는 TaskManager 애플리케이션을 구축합니다. 이 앱은 고품질이며 확장 가능하고, 상용 수준의 전환이 가능해야 합니다.

## 2. 핵심 요구사항
- **프레임워크:** Flutter (최신 안정 버전)
- **멀티 플랫폼 및 반응형 디자인:**
  - **지원 플랫폼:** 모바일(iOS/Android) 앱과 웹(Web) 브라우저에서 모두 완벽하게 동작해야 합니다.
  - **반응형 UI/UX:** 핸드폰, 태블릿, PC 등 모든 스크린 크기에서 어색함이 없는 레이아웃을 제공해야 합니다.
  - **적응형 레이아웃:** 화면 크기에 따라 컴포넌트 배치나 네비게이션 방식(예: Bottom NavBar vs Navigation Rail)이 자연스럽게 전환되는 깔끔한 디자인을 지향합니다.
- **제외 사항:** API 명세 중 **Admin(관리자) 관련 기능은 구현 범위에서 제외**합니다. (일반 사용자 기능에 집중)
- **아키텍처 및 재사용성:**
  - **Clean Architecture** (Data/Domain/Presentation 레이어)를 준수하여 관심사를 분리합니다.
  - **코드 재사용성:** UI 컴포넌트(Atoms, Molecules 등), 유틸리티 함수, API 클라이언트 로직 등을 모듈화하여 다른 프로젝트나 기능에서도 쉽게 재사용할 수 있도록 설계합니다.
  - **확장성:** 기능을 쉽게 추가하거나 삭제할 수 있도록 코드가 독립적으로 구성되어야 합니다.
  - **추상화:** 비즈니스 로직을 데이터 소스와 분리하기 위해 **Repository 패턴**을 사용합니다.
- **마이그레이션 고려 설계:**
  - 현재 서버(Render/Supabase)에서 **AWS**(예: Lambda + API Gateway 또는 App Runner) 등 다른 클라우드 환경으로 최소한의 코드 수정만으로 이전할 수 있도록 구성을 유연하게 설계해야 합니다.
- **지연 시간 처리 (무료 티어 최적화):**
  - **UX:** 견고한 **로딩 상태 표시(Shimmers/Spinners)** 및 **스플래시 화면**을 구현합니다.
  - **회복 탄력성:** 서버 응답이 느릴 경우를 대비해 **타임아웃 처리**, **재시도 로직 (Exponential Backoff)**, 명확한 **에러 메시지**를 구현합니다.
  - **캐싱:** 백엔드가 깨어나는 동안에도 앱이 기민하게 느껴지도록 로컬 스토리지(`shared_preferences` 또는 `hive`)를 사용하여 데이터를 캐싱하고 "snappy"한 사용감을 제공합니다.

## 4. 기술 스택 선호도
- **상태 관리:** Riverpod (확장성을 위해 권장)
- **HTTP 클라이언트:** Dio (인터셉터 및 재시도 로직 구현 용이)
- **로컬 DB:** Hive 또는 보안이 필요한 데이터의 경우 Secure Storage 사용.

## 5. 개발 워크플로우
1. V2 API 명세서와 프로토타입을 **분석**합니다.
2. Clean Architecture에 따라 프로젝트 구조를 **설정(Setup)**합니다.
3. 견고한 에러 및 지연 처리가 포함된 **데이터 레이어를 구현**합니다.
4. 프로토타입 디자인에 맞춰 **UI를 구축**합니다.
5. V2 백엔드에 대해 모든 CRUD 작업이 정상 작동하는지 **검증**합니다.
